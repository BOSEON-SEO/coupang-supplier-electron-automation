# 투비 쿠팡반출 양식 — WMS 연동 명세

> **버전** v1.0
> **대상** 투비 WMS 개발팀
> **목적** 쿠팡 자동화 앱과 WMS 사이의 양식 왕복 (다운로드 → WMS 작업 → 업로드) 시
> 데이터 손실/뻥튀기 없이 정확히 반영되도록 양식 구조와 채움 규칙을 합의한다.

---

## 1. 흐름

```
[자동화 앱]
  ① 쿠팡반출 양식 생성 (다운로드)         ┐
  ⇩                                        │
[WMS 앱]                                   │  같은 양식 파일을 양쪽에서 만들고 읽음
  ② 양식 입력 → 박스/송장 채워서 출력      │  (round-trip)
  ⇩                                        │
[자동화 앱]                                ┘
  ③ 양식 업로드 → 우리 데이터에 반영
```

핵심:

- **양식은 단일 시트 xlsx** (시트명 `반출`).
- **1행 = (물류센터, 발주번호, SKU Barcode, 박스/파렛트) 단위 한 건**.
- **같은 SKU 가 여러 박스/파렛트로 분할되면 행을 추가** (식별 키는 동일, 박스 정보만 다름).
- **다운받은 그대로 업로드 = 변경 0건 (idempotent)**.

## 2. 파일 구조

### 2.1 시트

- 단일 시트, 시트 이름 `반출`
- 1행 = 헤더, 2행부터 데이터

### 2.2 컬럼 (총 16개, 순서)

| # | 헤더 | 컬럼 종류 | 편집 가능 | 비고 |
|---|---|---|---|---|
| 1 | 물류센터 | 식별 키 | **금지** | 모든 행 동일 (그룹 식별) |
| 2 | 총 주문수량 | 센터 요약 | **금지** | 센터의 첫 행에만, 나머지 빈 셀 |
| 3 | 총 풀필반출 | 센터 요약 | **금지** | 센터의 첫 행에만, 나머지 빈 셀 |
| 4 | 발주번호 | 식별 키 | **금지** | 모든 행 동일 (그룹 식별) |
| 5 | 상품코드 | 식별 키 | **금지** | 모든 행 동일 (그룹 식별) |
| 6 | SKU Barcode | 식별 키 | **금지** | 모든 행 동일 (그룹 식별) |
| 7 | 상품명 | SKU 단위 | **금지** | 첫 분할 행에만, 나머지 빈 셀 |
| 8 | 신청수량 | SKU 단위 | **금지** | 우리가 창고에 보낼 SKU 총량 |
| 9 | 반출 | SKU 단위 | ✏️ 가능 | 풀필먼트에서 출고된 양 (SKU 총) |
| 10 | 창고수량 | SKU 단위 | ✏️ 가능 | 창고에서 실제 나간 양 (SKU 총) |
| 11 | 확정수량 | **박스 단위** | ✏️ 가능 | **그 박스/파렛트의 적재량** (행마다 다름) |
| 12 | 운송방법 | 박스 단위 | ✏️ 가능 | `쉽먼트` 또는 `밀크런` |
| 13 | 박스번호 | 박스 단위 | ✏️ 가능 | 1, 2, 3, … (쉽먼트일 때) |
| 14 | 송장번호 | 박스 단위 | ✏️ 가능 | 쉽먼트의 박스별 송장 |
| 15 | 파렛트번호 | 박스 단위 | ✏️ 가능 | 밀크런일 때 사용 |
| 16 | 비고 | SKU 단위 | ✏️ 가능 | 자유 메모 (재고없음, 부족사유 등) |

**컬럼 종류 정리**:

- **식별 키** — 우리 앱의 PO 행과 매칭하는 키. 절대 수정 금지.
- **센터 요약** — 정보 표시용. 자동 계산되므로 수정 금지.
- **SKU 단위** — SKU 전체에 1번 표시되는 값.
- **박스 단위** — 박스/파렛트 한 건마다 다른 값.

## 3. 분할 행 규칙

### 3.1 한 SKU = 한 행 (분할 없음)

가장 단순한 케이스. 그 SKU 가 박스 1개에 다 담김.

| 물류센터 | 발주번호 | 상품코드 | SKU Barcode | 상품명 | 신청수량 | 반출 | 창고수량 | 확정수량 | 운송방법 | 박스번호 |
|---|---|---|---|---|---|---|---|---|---|---|
| 호법 | 129868291 | 42248566 | 4549292221473 | TS7790A | 4 | 0 | 4 | 4 | 쉽먼트 | 1 |

### 3.2 한 SKU = 여러 박스로 분할

같은 SKU 가 박스 4개로 나뉘어 갈 때:

| 물류센터 | 발주번호 | 상품코드 | SKU Barcode | 상품명 | 신청수량 | 반출 | 창고수량 | **확정수량** | 운송방법 | **박스번호** | **송장번호** |
|---|---|---|---|---|---|---|---|---|---|---|---|
| 안성5 | 129751864 | 42248566 | 4549292221473 | TS7790A | 4 | 0 | 4 | **1** | 쉽먼트 | **1** | inv-001 |
| 안성5 | 129751864 | 42248566 | 4549292221473 | (빈) | (빈) | (빈) | (빈) | **1** | 쉽먼트 | **2** | inv-002 |
| 안성5 | 129751864 | 42248566 | 4549292221473 | (빈) | (빈) | (빈) | (빈) | **1** | 쉽먼트 | **3** | inv-003 |
| 안성5 | 129751864 | 42248566 | 4549292221473 | (빈) | (빈) | (빈) | (빈) | **1** | 쉽먼트 | **4** | inv-004 |

규칙:

1. **식별 키 4개 (물류센터/발주번호/상품코드/SKU Barcode)** 는 모든 행에 반복.
2. **SKU 단위 컬럼 (상품명/신청수량/반출/창고수량/비고)** 은 **첫 분할 행에만 적고**, 후속 행은 **빈 셀**.
3. **박스 단위 컬럼 (확정수량/운송방법/박스번호/송장번호/파렛트번호)** 은 **행마다 다른 값**.
4. **합계 검증**: `SUM(분할 행의 확정수량) == 신청수량` 이어야 함 (불일치 시 우리 앱이 경고).

### 3.3 같은 센터 안 여러 SKU

센터의 첫 행에만 `총 주문수량` / `총 풀필반출` 표기. 나머지 행은 빈 셀.

| 물류센터 | 총 주문수량 | 총 풀필반출 | 발주번호 | … |
|---|---|---|---|---|
| 안성4 | 196 | 0 | 129799598 | … (MG3090 4개 분할) |
| 안성4 | (빈) | (빈) | 129799598 | … |
| 안성4 | (빈) | (빈) | 129799598 | … |
| 안성5 | 4 | 0 | 129751864 | … (TS7790A 4박스 분할) |
| 안성5 | (빈) | (빈) | 129751864 | … |

## 4. 식별 키 (매칭 규칙)

업로드 시 우리 앱은 다음 4개 컬럼으로 행을 매칭합니다:

| 키 | 타입 | 설명 |
|---|---|---|
| 물류센터 | string | 그대로 비교 (한글 포함) |
| 발주번호 | string | 9~10자리 숫자, 그대로 비교 |
| 상품코드 | string | 자사 product_code (예: `4185634`) |
| SKU Barcode | string | 13자리 EAN 등 |

**중요**: 위 4개 컬럼은 **절대 수정/공백/추가/삭제하지 마세요**. 매칭 실패 시 그 행은 무시됩니다.

---

## 5. WMS 가 양식을 출력할 때 (= 우리에게 보내줄 때) 지킬 룰

### 5.1 행 분할

- WMS 가 박스 배정을 마친 후, **(센터 + 발주번호 + SKU Barcode + 박스번호) 단위로 1행** 출력.
- 한 SKU 가 여러 박스로 나뉘면 위 §3.2 패턴 그대로.

### 5.2 빈 셀 vs 채움 셀

- **SKU 단위 컬럼은 분할 행 중 첫 행에만** 채우고 나머지는 빈 셀로 두기를 권장.
- 모든 행에 같은 값을 복사해도 우리 앱은 **first-wins** 로 처리 (= 첫 비어있지 않은 값 사용) 하므로 안전. 단 시각적으로 어수선해짐.
- **확정수량 (11번)** 은 **반드시 행마다 그 박스의 적재량** 만 적기. 같은 값 복사 시 SUM 으로 부풀려져 잘못된 결과.

### 5.3 송장번호

- 쉽먼트 박스에 부여된 운송사 송장번호. 박스마다 다름.
- 같은 박스번호의 행이 여러 개라면 (= 한 박스에 여러 SKU) 모두 같은 송장번호를 적어주세요.

### 5.4 운송방법

- 값은 정확히 `쉽먼트` 또는 `밀크런` 둘 중 하나. (한글 그대로, 공백 없음)
- 다른 값이 들어오면 우리 앱이 무시.

### 5.5 박스번호 / 파렛트번호

- 쉽먼트일 때 → 박스번호만 채움 (1, 2, 3, … 양의 정수). 파렛트번호 빈칸.
- 밀크런일 때 → 파렛트번호만 채움 (예: `P1`, `P2` 또는 임의 라벨). 박스번호 빈칸.

### 5.6 ❗ 권고

- **신청수량 컬럼 수정 금지** — 우리가 보낸 SKU 총량 그대로 두면 됨. 만약 수정하면 우리 앱이 그 값으로 PO 를 덮어씀.
- **창고수량 / 반출 / 비고** 는 자유롭게 수정 가능.
- 새 행 추가 시: **식별 키 4개** 가 우리 앱의 PO 에 존재해야 함 (없는 키는 무시).

## 6. 우리 앱이 받은 값을 어떻게 쓰는지 (참고)

업로드된 양식의 값은 다음 위치에 반영됩니다:

| 양식 컬럼 | 우리 앱 반영 위치 |
|---|---|
| 신청수량 | confirmation.xlsx · po.xlsx · po-tbnws.xlsx 의 **확정수량** (3 파일 동시 sync) |
| 반출 | po-tbnws.xlsx 의 **반출수량** |
| 창고수량 | 내부 스냅샷 (`coupang-export.json`) — 참고용으로만 보관 |
| 확정수량 | 운송분배 (`transport.json`) 의 **lots[].items[].qty** (박스별 적재량) |
| 운송방법 | 운송분배 lot 의 type (쉽먼트/밀크런) |
| 박스번호 | 운송분배 lot 의 box index |
| 송장번호 | 운송분배 lot 의 boxInvoices[box-1] |
| 파렛트번호 | 운송분배 lot 의 items[].palletNo |
| 비고 | 운송분배 lot 의 skuNotes[rowKey] |

**부속 효과**:

- 신청수량을 줄여서 보내주면 → 자동으로 발주확정서의 **납품부족사유** 가 채워짐 (신청 < 원래 발주수량 일 때).
- 합계 검증 (`SUM(확정수량) == 신청수량`) 이 안 맞으면 우리 앱이 경고 alert 표시 (반영은 진행).

## 7. 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 업로드 후 일부 행이 반영 안 됨 | 식별 키 4개 중 하나가 변경/공백 | 다운로드한 그대로 두고 재시도 |
| SKU 총 수량이 N배로 부풀려짐 | 확정수량을 모든 분할 행에 같은 값으로 복사 | 박스별로 다른 적재량을 입력 (또는 빈 셀로) |
| 박스/송장번호 매칭 안 됨 | 운송방법 컬럼 값이 `쉽먼트` 또는 `밀크런` 이 아님 | 한글 정확히 두 단어 중 하나로 |
| "수량 불일치 N건" 경고 | SUM(확정수량) ≠ 신청수량 | 박스별 합이 SKU 총량과 같은지 확인 |
