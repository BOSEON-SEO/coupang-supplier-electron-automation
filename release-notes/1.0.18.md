## 1.0.18 — 운송분배·쿠팡반출 신뢰성 + UX 정리

### 기능
- 투비 쿠팡반출 모달에 **🗑 설정 초기화** 버튼 — transport.json / 스냅샷 / 양식 캐시를 한 번에 비우고 처음부터 다시 시작
- 웹뷰 우측 패널 **드래그 리사이저** — 좌측 가장자리 핸들로 폭 조정(320–1200px), localStorage 영속, 중앙 그립 인디케이터 hover/active 강조

### 버그
- 같은 SKU 가 다른 발주번호에 동시 존재할 때 쿠팡반출 업로드 매칭이 두 PO 를 한 그룹으로 합쳐서 transportEntries 가 통째 중복되던 문제 — lookup 키에 발주번호 포함, 분할행 발주번호 last-seen 보강
- 쿠팡반출 apply 가 transport.json 을 v4 raw 로만 저장해 milkrun_register.py / shipment_register.py 가 처리할 entry 0건으로 끝나던 문제 — v3 호환 필드(transportType 등) flatten 동시 저장
- 운송분배 기존 진입 시 자동으로 빈 쉽먼트 lot 1개 생성되던 동작 제거 — 사용자가 직접 추가하거나 엑셀 업로드로 채워짐
- 밀크런 팔레트 자동 채움 조건(`existing.length < maxPallet`) 이 maxPallet=1 케이스를 못 잡던 버그 — `allEmpty` 만으로 판정
- 자동 로그인이 활성 Keycloak/xauth 페이지를 무시하고 stale supplier.coupang.com 페이지로 세션 유효 false-positive 내던 문제 — find_vendor_page 우선순위에서 활성 로그인 페이지를 최우선으로

### UX
- 밀크런/쉽먼트 등록 클릭 시 등록 대상이 없으면 카운트다운 스킵하고 즉시 토스트로 안내 (Python 스크립트의 transportType 기준과 일치)
- 카운트다운 모달 동안에는 웹뷰 패널이 자동으로 펼쳐지지 않음 — 카운트다운 종료 후(Python 실행 시작 시점) 펼침. 이전엔 BrowserView 가 모달 위로 overlay 되며 스타일 깨지던 이슈
- 카운트다운 모달 z-index 99999 — FortuneSheet 스크롤바/툴바가 dim 위로 비치던 문제 해결
