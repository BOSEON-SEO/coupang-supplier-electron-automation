# 쿠팡 서플라이어 자동화 앱 — 디자인 브리프

> 디자이너에게 전달용 단일 문서.
> "무슨 앱인지 / 무엇을 해야 하는지 / 어떤 화면이 있는지 / 다룰 때 주의할 것" 을 한 번에 정리.

---

## 1. 한 줄 요약

쿠팡 공급사(셀러) 가 매일 반복하는 **PO 다운 → 발주 확정 → 운송 분배 → 양식 업로드** 일련의 작업을, 우리 앱이 쿠팡 사이트를 자동 조작해서 대신 해주는 데스크톱 앱. 사람은 중간중간 검수/수정만 한다.

대상 사용자: 쿠팡 셀러 운영팀 (반복 업무에 지친 직원). 하루 수십 건의 발주를 처리.

---

## 2. 도메인 빠른 사전

| 용어 | 뜻 |
|---|---|
| **벤더** | 쿠팡과 거래하는 회사(셀러) 1개. 우리 앱은 멀티벤더 지원. 예: 캐논, 키크론. |
| **발주(PO)** | 쿠팡이 셀러에게 "이 SKU 몇 개 보내달라" 라고 요청하는 단위. 발주번호로 식별. |
| **차수** | 같은 날짜에 발주가 여러 건이면 1차/2차/3차... 로 구분. |
| **물류센터** | 쿠팡 입고 창고. 한 발주는 여러 물류센터로 분산 입고됨 (예: 안성4, 화성2, 덕평). |
| **확정수량** | 셀러가 "이만큼 보내겠다" 확정한 수량. (쿠팡 요청 ≥ 확정 가능) |
| **운송방법** | `쉽먼트`(택배) 또는 `밀크런`(파레트 단위 트럭). 센터별로 다름. |
| **박스/팔레트** | 쉽먼트는 박스 단위. 밀크런은 팔레트 안에 박스 여러 개. |
| **WMS** | 외부 시스템 — 창고 관리. 우리 앱과 엑셀로 데이터 주고받음. |

## 3. 사용자가 매일 하는 일 (메인 플로우)

하루 작업 = **하나의 "차수"**. 캘린더 → 차수 더블클릭 → 작업뷰 진입.

```
[캘린더 화면]
   └─ 오늘 날짜에 점등된 벤더 목록 → 차수 카드 클릭
        ↓
[작업뷰] — 4단계 진행
   1️⃣ PO 다운     : 쿠팡 사이트에서 SKU 목록 엑셀 다운
   2️⃣ 발주 확정    : SKU 별 "보냄/반려" 결정 + 확정수량 입력 → 쿠팡에 업로드
   3️⃣ 물류 처리    : 센터별로 박스/팔레트/송장 분배
   4️⃣ 완료        : 차수 마감
```

각 단계는 `phase-stepper` 로 시각화. 위험한 동작(쿠팡에 실제 업로드) 직전엔 **3초 카운트다운 + 취소 버튼**.

## 4. 화면(뷰) 인벤토리

### 4.1 메인 윈도우 — 좌우 2분할

```
┌─────────────────────────────────────────────┐
│  헤더 (벤더 선택 / 라이선스 상태 / 업데이트) │
├──────┬──────────────────────────┬───────────┤
│ 사이 │  메인 영역 (작업/달력/   │  웹뷰     │
│ 드바 │   설정/플러그인 뷰)       │  (슬라이드│
│      │                          │   패널)   │
│      │                          │           │
└──────┴──────────────────────────┴───────────┘
```

- **사이드바 (좌)** — 달력 / 작업 / 설정 / 플러그인 메뉴 토글. 64px 폭.
- **메인 영역 (가운데)** — 현재 모드(달력/작업/설정/플러그인)에 따라 화면 통째로 전환.
- **웹뷰 (우, 슬라이드)** — 쿠팡 사이트가 떠 있는 패널. 평소엔 접혀있고 자동화 동작 시 자동 펼침. 좌측 가장자리 드래그로 폭 조정 (320–1200px). 토글 버튼으로 수동 펼침/접힘.

### 4.2 달력 뷰 (Calendar View)

월 단위 달력. 날짜 칸 안에:
- 그 날 진행 중/완료된 차수 카드 (벤더 색상별)
- 각 카드: `벤더명 · N차 · 진행단계`
- 클릭 → 작업뷰 진입
- 우측 상단에 "원격 달력 동기화" 인디케이터 (외부 일정 fetch)

**디자인 포인트**: 카드가 매일 평균 3–10개. 한눈에 진행률(어디까지 갔는지) 보이게.

### 4.3 작업 뷰 (Work View) ★ 가장 중요

상단 헤더:
- 달력으로 돌아가기 / 벤더·날짜·차수 표시 / **PhaseStepper(4단계 진행 동그라미)** / 웹뷰 토글 / 작업완료 버튼

하단 본체 — **탭 구조**:

| 탭 | 내용 |
|---|---|
| **PO 원본** | 쿠팡에서 다운받은 SKU 목록 (Excel-like 그리드, 셀 편집 가능) |
| **확정서** | 쿠팡 업로드용 발주확정 양식 (SKU별 보냄/반려/확정수량/입고유형/부족사유) |
| **운송분배** | 센터별 박스·팔레트·송장 카드. 가장 복잡. 별도 4.4 항목. |
| **결과/출력** | 다운로드된 파일 모아보기, 통합 양식 등 |
| **(플러그인 탭들)** | 활성화된 벤더별 추가 기능 (예: 캘린더 동기화, 이플렉스 출고, 투비 쿠팡반출) |

**디자인 포인트**:
- 탭 전환이 잦음. 키보드 단축키도 고려.
- 각 탭 내부에 행이 수십~수백. 검색/필터 필수.
- 셀 편집 UX 가 핵심. 한글 IME 안정성 중요.

### 4.4 운송분배 뷰 (Transport View) — 가장 복잡한 화면

물류센터별 카드 1개씩:

```
┌────────────────────────────────────────────┐
│ 📦 안성4 (밀크런) — 총 192개                │
├────────────────────────────────────────────┤
│  출고지: [드롭다운: 평택공장]               │
│  총 박스 수: [입력]                         │
│                                            │
│  ─ 팔레트 1 (T11, 48 박스)                 │
│  ─ 팔레트 2 (T11, 48 박스)                 │
│  [+ 팔레트 추가]                            │
│                                            │
│  ─ SKU 분배:                                │
│    캐논 잉크젯 4549292062830                │
│      └ 팔레트 1: 48개  / 팔레트 2: 48개   │
│  [+ 쉽먼트 추가] [+ 밀크런 추가]           │
└────────────────────────────────────────────┘
```

- **lot 단위**: 한 센터에 쉽먼트 lot 0+개, 밀크런 lot 0+개 공존 가능.
- 쉽먼트 lot: 박스번호별 SKU 분배 + 송장번호.
- 밀크런 lot: 팔레트(프리셋·박스수) + 팔레트 내 SKU 분배.
- 상단에 **`전체/쉽먼트/밀크런` 필터 칩**.

**디자인 포인트**:
- 데이터 입력이 많음 — 키보드 네비, 자동 채움이 핵심.
- 엑셀(WMS 결과) 업로드 한 번으로 통째 자동 채워짐.
- 위험: 빈 lot 잔존, 같은 SKU 가 여러 박스에 split 되는 것 시각적으로 표현 필요.

### 4.5 설정 뷰 (Settings)

- 벤더 추가/삭제/수정
- 자격증명(쿠팡 ID/PW) 보관
- 출고지 / 팔레트 프리셋 / 기본 운송방법 등 마스터 데이터
- 라이선스 상태 (만료 임박 배너)
- 플러그인 on/off

### 4.6 플러그인 뷰

벤더별 특화 기능. 현재 캐논(`tbnws` 플러그인) 만 활성:
- **이플렉스 출고** — 외부 시스템에 출고 요청
- **재고이동 등록**
- **출고예정 등록**
- **투비 쿠팡반출** — WMS 결과 엑셀(16컬럼) 받아서 우리 데이터에 반영 + 쿠팡 업로드용 양식 생성. **모달**로 제공 중.

### 4.7 모달들

자주 뜨는 모달:
- **카운트다운 확인 모달** — 위험 동작(쿠팡 업로드) 직전 3초.
- **확정서/운송분배 편집 모달**
- **투비쿠팡반출 모달** — 미리보기 + 양식 다운/업로드.
- **에러/토스트** — 우상단에서 슬라이드.

## 5. 인터랙션 패턴

| 패턴 | 어디서 | 디자인 메모 |
|---|---|---|
| **Excel-like 그리드 편집** | PO원본/확정서/결과 탭 | 더블클릭/Enter/F2 로 편집, ↑↓←→ 네비, 한글 IME 처리 |
| **자동화 카운트다운** | 쿠팡에 실제 업로드 직전 | 3초 + 취소 + 진행률 |
| **웹뷰 슬라이드 자동 펼침** | Python 자동화 실행 시 | 사용자가 동작 검증 가능 |
| **드래그 리사이즈** | 웹뷰 패널 폭 | 좌측 핸들(14px), localStorage 영속 |
| **드롭존 업로드** | 운송분배/투비쿠팡반출 | xlsx 파일 드래그 or 파일 선택 |
| **로그 스트리밍** | 자동화 실행 중 | 작업뷰 하단 로그 패널, 색상별 INFO/ERROR |

## 6. 시각 정체성 (현재 상태)

- 색상: 쿠팡 브랜드와 무관한 중립 톤. 파랑(#2563eb) primary.
- 다크모드 미지원.
- 폰트: 시스템 기본.
- 아이콘: 이모지 사용 중 (📅📦📊✓ 등) — 정식 아이콘셋 도입 필요.
- 지표 시각화 부족 — 진행률, 차수별 상태 한눈에 보기 어려움.

**개선 여지**:
- 정보 밀도가 높은 화면(특히 운송분배) 의 가독성
- 복잡한 폼 입력의 단축키/자동완성 UX
- 한 작업의 단계별 시각화 (스테퍼 + 시각적 흐름도)
- 빈 상태/에러 상태 일관성

## 7. 기술적 제약 (디자인이 알아야 할 것)

- **Electron + React** — 브라우저 기반이라 보통의 웹 디자인 스킬 그대로 사용 가능.
- **윈도우 안에 또 다른 브라우저 (BrowserView/WCV)** — 쿠팡 사이트가 메인 윈도우 위에 native overlay 됨. 이 영역은 **DOM 위에 덮여 있어 우리 UI 가 못 닿음**. 슬라이드 패널 안에서만 살아있게 설계됨.
- **데이터 로컬 파일** — 모든 작업 산출물은 `%LOCALAPPDATA%/CoupangAutomation/{date}/{vendor}/{seq}/` 에 xlsx/json 파일로 저장. 클라우드 X.
- **Python subprocess** — 자동화 로직은 Python(Playwright). 디자인 무관, but 실행 중에는 로그 스트림이 작업뷰 하단에 흐름.
- **다국어**: 한국어 only.

## 8. 사용자 페르소나 / 페인 포인트

**주 사용자**: 쿠팡 운영 직원, 30~40대, 매일 5–8시간 이 앱 사용.

**현재 불만 (실제 피드백)**:
- "확정서 셀 편집 시 한글 IME 깨짐"
- "운송분배 화면이 너무 복잡, 스크롤이 길다"
- "WMS 엑셀 업로드 후 빈 팔레트 카드가 자동 안 채워짐"
- "웹뷰가 화면을 너무 차지 (해결됨 — 슬라이드 패널화)"
- "투비쿠팡반출 모달의 미리보기 테이블이 좁음"
- "결과 탭이 잡탕 — 무엇이 들어있는지 한눈에 안 보임"

**칭찬받는 부분**:
- 자동화 카운트다운 + 웹뷰 실시간 확인
- 캘린더 뷰
- 다중 벤더 격리

## 9. 디자이너에게 요청하고 싶은 것

원하시는 만큼 자유롭게 가능하지만, 우선순위:

1. **운송분배 뷰 재디자인** — 데이터 밀도 ↓, 키보드 효율 ↑.
2. **작업 진행률의 시각화** — 캘린더에서, 작업뷰 헤더에서, 그리고 가능하면 별도 대시보드 화면.
3. **셀 편집 그리드의 시각 언어** — 편집/저장/에러 상태, hover/selection.
4. **빈 상태 / 에러 상태 / 로딩 상태** 의 일관된 패턴.
5. **아이콘 시스템** — 이모지 → 정식 셋(예: Lucide).
6. **다크모드** — 향후.

## 부록 — 데이터 모델 한눈에

```
job (= 차수)
├─ date · vendor · sequence
├─ phases: [PO다운, 발주확정, 물류처리, 완료]
├─ confirmation.xlsx     (확정서)
├─ po.xlsx / po-tbnws.xlsx (원본 PO)
├─ transport.json        (운송분배)
│    └─ assignments[wh] = { lots: [
│         { type: '쉽먼트', boxCount, boxInvoices[], items[{rowKey,qty,boxNo}] }
│         { type: '밀크런', pallets[{presetName, boxCount}], items[{rowKey,qty,palletNo}] }
│       ] }
└─ coupang-export.json   (투비쿠팡반출 스냅샷)
```

플러그인이 추가하는 데이터:
- `manifest.pluginData.tbnws.eflexRequested` 등
